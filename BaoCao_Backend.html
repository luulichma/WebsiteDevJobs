BÁO CÁO KỸ THUẬT: PHÂN HỆ BACK-END
Dự án: Nền tảng Tuyển dụng DevJobs (RESTful Web API)

Lưu ý: Phân hệ Back-end này được xây dựng theo mô hình RESTful Web API (API-only), KHÔNG PHẢI là mô hình Web MVC truyền thống (trả về giao diện HTML/Razor Views). Backend chỉ nhận Request và trả về dữ liệu định dạng JSON, còn việc vẽ giao diện hoàn toàn do Frontend (ReactJS) đảm nhận.

1. TỔNG QUAN PHÂN HỆ BACK-END

Phân hệ Back-end của hệ thống DevJobs đóng vai trò là lõi xử lý nghiệp vụ trung tâm, chịu trách nhiệm giao tiếp với cơ sở dữ liệu (Database), thực thi bảo mật, kiểm soát quyền truy cập và cung cấp dữ liệu cho các ứng dụng phía Client (Frontend Web/Mobile) thông qua các giao thức HTTP dưới dạng cấu trúc dữ liệu JSON.

Hệ thống được thiết kế theo kiến trúc Client-Server độc lập, trong đó Back-end hoạt động hoàn toàn dưới dạng RESTful Web API, nhằm tách biệt hoàn toàn khỏi logic hiển thị giao diện, giúp tối ưu hóa khả năng mở rộng.

2. CÔNG NGHỆ VÀ NGĂN XẾP (TECH STACK)

Hệ thống sử dụng các công nghệ hiện đại chuyên biệt để tối ưu hóa hiệu suất ứng dụng phân tán:

  • Ngôn ngữ lập trình: C# (C-Sharp).
  • Web Framework: ASP.NET Core Web API (.NET 10.0). Đây là framework hiệu suất cực cao của Microsoft chuyên dùng để xây dựng các dịch vụ HTTP.
  • Cơ sở dữ liệu (DBMS): Microsoft SQL Server 2022 (Sẵn sàng nền tảng để triển khai môn học Phân tán cơ sở dữ liệu trong tương lai).
  • O/RM (Object-Relational Mapping): Entity Framework Core 10.0 thiết lập theo phương pháp Database-First (Tự động sinh Model từ kịch bản quy chuẩn thiết kế CSDL thực tế).
  • Bảo mật & Xác thực:
    • JSON Web Token (JWT): Thiết lập cơ chế Auth dạng Bearer Token (Stateless).
    • BCrypt.Net: Mã hóa một chiều (Hashing) mật khẩu người dùng trước khi lưu xuống CSDL.
  • Tài liệu lập trình hóa (API Docs): Swashbuckle / Swagger UI.

3. KIẾN TRÚC VÀ CÁC THÀNH PHẦN (MODULES) CỐT LÕI

3.1. Design Patterns (Mẫu Thiết kế) áp dụng

  • Controller Pattern: Lớp trung gian tiếp nhận HTTP Request (GET, POST, PUT, DELETE), xác thực tham số và điều hướng xuống lớp xử lý dữ liệu.
  • DTO (Data Transfer Object) Pattern: Tách biệt hoàn toàn bản ghi dữ liệu Model (của Entity Framework) và luồng dữ liệu trả về cho Front-end. Ngăn chặn triệt để rò rỉ dữ liệu nhạy cảm (như mục PasswordHash) và giải quyết triệt để lỗi vòng lặp tham chiếu JSON (Circular Reference).
  • Dependency Injection (DI): Được tích hợp sẵn trong .NET, giúp đăng ký tự động và quản lý vòng đời của AppDbContext, JwtService qua Program.cs.

3.2. Chức năng phân rã Modules (API Controllers)

Hệ thống Back-end được chia làm 6 bộ điều khiển (Controllers) chính, phản ánh trực tiếp các thực thể và chức năng trong đồ án:

  1. Auth Module (Đăng nhập/Đăng ký): Cung cấp endpoint đăng ký mới và đăng nhập. Khi xác thực thành công, trả về JWT Token chứa thông tin mã hóa (UserId, Role, CompanyId) giúp Frontend định danh phiên hoạt động suốt quá trình sử dụng.
  2. Users Module (Người dùng): Xử lý nghiệp vụ cá nhân hóa (Profile) cho từng tài khoản. Bên cạnh đó, hỗ trợ hệ thống Admin các chức năng quản lý từ xa như Get All, Khóa (Suspend), hoặc Xóa (Delete) tài khoản của ứng viên/công ty cố tình vi phạm.
  3. Jobs Module (Tin tuyển dụng):
    • Góc nhìn Khách/Ứng viên: Lấy danh sách việc làm công khai với tham số lọc truy vấn (Tìm kiếm theo tên, lương, vị trí, kỹ năng...). API tích hợp sẵn thuật toán phân trang (Pagination).
    • Góc nhìn Recruiters: Đăng tin tuyển dụng (mặc định cần chờ Admin duyệt), sửa nội dung, tắt tin và gia hạn tin (renew).
  4. Applications Module (Hồ sơ ứng tuyển): Là cầu nối giữa Candidate và Job. Xử lý nghiệp vụ nhận CV ứng tuyển, và cung cấp bảng điều khiển để nhà tuyển dụng cập nhật luồng trạng thái ứng viên (Từ "Chờ xem xét" -> "Đang phỏng vấn" -> "Chấp nhận"/"Từ chối").
  5. Companies Module (Hồ sơ Doanh nghiệp): Cung cấp thông tin tập đoàn, trụ sở và logo hiển thị.
  6. Skills Module (Danh mục Chuyên môn): Cung cấp từ điển các kỹ năng ngành nghề (IT, Marketing,...) để phân rã chuyên môn dễ dàng.

4. BẢO MẬT VÀ QUẢN LÝ DATABASE

Phương thức giao tiếp Cơ sở dữ liệu

Quá trình map cơ sở dữ liệu tuân thủ nghiêm ngặt chuẩn hệ cơ sở dữ liệu quan hệ Database-First. Cơ sở dữ liệu DevJobs được thiết kế chặt chẽ trên cấu trúc SQL Server Management Studio (SSMS). Backend sử dụng lệnh Scaffold-DbContext để phân tích cấu trúc khóa cứng và sinh tự động các Entity C#.

Công nghệ EF Core đã xử lý tự động và hoàn hảo quan hệ Nhiều-Nhiều (Many-to-Many), bóc tách quan hệ Job-Skills thành mảng Collection cấp ứng dụng (trong Memory) mà không đòi hỏi thao tác can thiệp trực tiếp từ lập trình viên xuống bảng nối trung gian.

Chính sách Phân quyền và Quy tắc kết nối (CORS)

  • CORS Policy (Cross-Origin Resource Sharing): Hệ thống được thiết lập chế độ bảo vệ nguồn gốc chặt chẽ. Chỉ cho phép đích danh các tên miền Front-end như localhost:5173 hoặc localhost:3000 truy cập API. Tránh tấn công đánh cắp dữ liệu chéo từ các máy chủ lạ.
  • Authorization Barrier (Rào chắn Ủy quyền): Mỗi phân đoạn Endpoint API đều gán rào chắn [Authorize(Roles="admin")], [Authorize(Roles="recruiter")],... Bất kỳ token JWT giả mạo nào gửi tới Controller cũng sẽ bị đá ra ngoài qua mã lỗi 401 Unauthorized hoặc 403 Forbidden trước cả khi câu query kịp chạm xuống hệ cơ sở dữ liệu SQL Server.

5. TỔNG KẾT

Phân hệ Back-end RESTful Web API của dự án DevJobs mang tới một giải pháp trung tâm tinh gọn, kiến trúc bền bỉ và bảo mật cao. Việc thiết kế hệ thống C# hoàn toàn độc lập với Frontend, kết hợp hệ quản trị cơ sở dữ liệu tiêu chuẩn Microsoft SQL Server 2022 giúp đảm bảo năng lực hệ thống dễ dàng cấu hình nhân bản và triển khai môn học hệ thống CSDL Phân Tán mà không cần làm tổn tác đến lõi mã nguồn API hiện tại.
