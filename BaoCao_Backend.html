📋 Báo cáo API Backend — DevJobs

Tài liệu mô tả toàn bộ REST API của nền tảng tuyển dụng IT DevJobs

ASP.NET Core 8.0 Entity Framework Core SQL Server JWT Auth Version 1.0

1. Giới thiệu dự án

DevJobs là nền tảng tuyển dụng IT cho phép ứng viên tìm kiếm việc làm, nhà tuyển dụng đăng tin và quản trị viên kiểm soát toàn bộ hệ thống. Backend được xây dựng theo kiến trúc REST API, hoàn toàn tách biệt khỏi frontend.

ASP.NET Core 8.0 Entity Framework Core 8 SQL Server BCrypt.Net-Next JWT Bearer Token Swagger / OpenAPI Google OAuth 2.0

Phân quyền người dùng (3 Role)

RoleTên gọiQuyền truy cập
candidateỨng viênTìm việc, nộp hồ sơ, quản lý hồ sơ cá nhân
recruiterNhà tuyển dụngĐăng tin, quản lý tin, xem & xử lý hồ sơ ứng viên
adminQuản trị viênDuyệt tin, quản lý tất cả người dùng, xem thống kê

4. Bảo mật & Xác thực

🔐 JWT Bearer Token

Mỗi token gồm 3 phần: Header (thuật toán HS256) · Payload (UserId, Role, CompanyId) · Signature. Token có thời hạn 7 ngày.

// Cấu hình JWT trong Program.cs
builder.Services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(options => {
        options.TokenValidationParameters = new TokenValidationParameters {
            ValidateIssuerSigningKey = true,
            IssuerSigningKey = new SymmetricSecurityKey(key),
            ValidateLifetime = true
        };
    });

🔑 BCrypt Password Hashing

Mật khẩu không bao giờ lưu dạng plaintext. Dùng BCrypt với cost factor mặc định (10 rounds). Không thể giải mã ngược.

// Hash khi đăng ký
user.PasswordHash = BCrypt.Net.BCrypt.HashPassword(req.Password);

// Verify khi đăng nhập
bool valid = BCrypt.Net.BCrypt.Verify(req.Password, user.PasswordHash);

🛡️ Phân quyền theo Role

DecoratorÝ nghĩa
[AllowAnonymous]Không cần đăng nhập
[Authorize]Phải đăng nhập (bất kỳ role)
[Authorize(Roles = "candidate")]Chỉ Ứng viên
[Authorize(Roles = "recruiter")]Chỉ Nhà tuyển dụng
[Authorize(Roles = "admin")]Chỉ Admin

🌐 CORS Policy

Chỉ cho phép request từ origin được whitelist. Cấu hình trong Program.cs:

builder.Services.AddCors(opt => opt.AddPolicy("AllowFrontend", p =>
    p.WithOrigins("http://localhost:5173", "http://localhost:5174")
     .AllowAnyMethod().AllowAnyHeader()));

5. API Endpoints

🔐 AuthController  Base: /api/auth
MethodEndpointAuthMô tả
POST /api/auth/login Public Đăng nhập bằng email/password, trả về JWT token
POST /api/auth/register Public Đăng ký tài khoản mới (candidate hoặc recruiter)
POST /api/auth/google Public Đăng nhập bằng Google OAuth — xác minh ID token với Google, tự tạo tài khoản nếu mới
💼 JobsController  Base: /api/jobs
MethodEndpointAuthMô tả
GET/api/jobsPublicDanh sách tin tuyển dụng (hỗ trợ filter: keyword, location, jobType, salary, skill, phân trang)
GET/api/jobs/{id}PublicChi tiết một tin tuyển dụng
GET/api/jobs/myRecruiterDanh sách tin đã đăng của nhà tuyển dụng hiện tại
GET/api/jobs/pendingAdminDanh sách tin đang chờ duyệt
POST/api/jobsRecruiterĐăng tin tuyển dụng mới (trạng thái pending, chờ admin duyệt)
PUT/api/jobs/{id}RecruiterCập nhật thông tin tin tuyển dụng
PUT/api/jobs/{id}/approveAdminPhê duyệt tin (chuyển status → active)
PUT/api/jobs/{id}/rejectAdminTừ chối tin (chuyển status → rejected)
PUT/api/jobs/{id}/closeRecruiterĐóng tin tuyển dụng (chuyển status → closed)
PUT/api/jobs/{id}/renewRecruiterGia hạn tin thêm 30 ngày (chuyển status → active)
📄 ApplicationsController  Base: /api/applications
MethodEndpointAuthMô tả

PUT/api/applications/{id}/statusRecruiterCập nhật trạng thái hồ sơ: pending → shortlisted → interview → accepted / rejected
👤 UsersController  Base: /api/users
MethodEndpointAuthMô tả

GET/api/usersAdminDanh sách toàn bộ người dùng (trừ admin)
PUT/api/users/{id}/suspendAdminKhóa / Mở khoá tài khoản người dùng
DELETE/api/users/{id}AdminXoá tài khoản người dùng (không thể xoá admin)
🏢 CompaniesController  Base: /api/companies

PUT/api/companies/myRecruiterCập nhật thông tin công ty của recruiter hiện tại

📖 Hướng dẫn test API trên Swagger

  1. Mở http://localhost:5050/swagger
  2. Vào POST /api/auth/login → Try it out → Điền email/password → Execute
  3. Copy chuỗi token từ Response
  4. Bấm nút 🔒 Authorize (góc trên phải) → Paste token → Authorize
  5. Gọi bất kỳ API nào — token sẽ tự động đính kèm vào header
