UC01 Đăng nhập & Phân quyền

a) ERD liên quan

erDiagram USERS { int user_id PK varchar email UK varchar password_hash varchar full_name varchar role varchar status datetime created_at }

UC01 chỉ truy cập bảng USERS để xác thực email/password và kiểm tra role + status.

b) API Endpoints

MethodEndpointAuthMô tả
POST/api/auth/loginPublicĐăng nhập bằng email/password → trả về JWT Token + user info
POST/api/auth/registerPublicĐăng ký tài khoản mới (candidate/recruiter)
POST/api/auth/googlePublicĐăng nhập bằng Google OAuth — tự tạo tài khoản nếu mới

c) Kết quả giao diện

Trang đăng nhập với form Email/Password và Google OAuth

Sau khi đăng nhập với role Candidate → chuyển đến trang Tìm việc

Sau khi đăng nhập với role Recruiter → chuyển đến trang Quản lý tin

Sau khi đăng nhập với role Admin → chuyển đến trang Duyệt tin

UC02 Tìm kiếm việc làm

a) ERD liên quan

erDiagram JOBS ||--o{ JOB_SKILLS : "có" SKILLS ||--o{ JOB_SKILLS : "thuộc" COMPANIES ||--o{ JOBS : "đăng" JOBS { int job_id PK int company_id FK varchar title text description varchar location varchar job_type decimal salary_min decimal salary_max varchar status } COMPANIES { int company_id PK varchar company_name varchar logo_url } SKILLS { int skill_id PK varchar skill_name UK } JOB_SKILLS { int job_id PK_FK int skill_id PK_FK }

UC02 truy vấn bảng JOBS kết hợp JOIN với COMPANIES và SKILLS (qua JOB_SKILLS) để lọc theo keyword, location, jobType, salary, skill. Hỗ trợ tìm kiếm tiếng Việt không dấu (Vietnamese_CI_AI Collation).

b) API Endpoints

MethodEndpointAuthMô tả
GET/api/jobs?keyword=&location=&jobType=&skill=&salaryMin=&salaryMax=&page=&pageSize=PublicTìm kiếm tin tuyển dụng với bộ lọc và phân trang. Hỗ trợ tìm tiếng Việt không dấu

c) Kết quả giao diện

Trang chủ DevJobs với thanh tìm kiếm và việc làm nổi bật

Trang tìm kiếm việc làm với bộ lọc Địa điểm, Loại hình và sắp xếp

UC03 Xem chi tiết & Ứng tuyển

a) ERD liên quan

erDiagram USERS ||--o{ APPLICATIONS : "nộp đơn" JOBS ||--o{ APPLICATIONS : "nhận đơn" JOBS { int job_id PK int company_id FK varchar title text description text requirements varchar status } APPLICATIONS { int application_id PK int job_id FK int candidate_id FK varchar cv_url text cover_letter varchar status datetime applied_at } USERS { int user_id PK varchar email varchar full_name varchar role }

UC03 đọc chi tiết từ JOBS, sau đó tạo bản ghi mới trong APPLICATIONS liên kết candidate_id (USERS) với job_id. Ràng buộc UNIQUE (job_id, candidate_id) đảm bảo không trùng.

b) API Endpoints

MethodEndpointAuthMô tả
GET/api/jobs/{id}PublicLấy chi tiết một tin tuyển dụng
POST/api/applicationsCandidateNộp hồ sơ ứng tuyển (kiểm tra trùng, job active)
GET/api/applications/my-applicationsCandidateLịch sử ứng tuyển của tôi

c) Kết quả giao diện

Trang chi tiết công việc với mô tả, yêu cầu, nút Ứng tuyển

Trang lịch sử ứng tuyển của ứng viên — hiển thị trạng thái hồ sơ

UC04 Quản lý hồ sơ cá nhân

a) ERD liên quan

erDiagram USERS { int user_id PK varchar email UK varchar full_name varchar phone varchar cv_url text skills varchar role datetime created_at }

UC04 đọc và cập nhật thông tin cá nhân trong bảng USERS (full_name, phone, cv_url, skills).

b) API Endpoints

MethodEndpointAuthMô tả
GET/api/users/meBất kỳLấy thông tin user đang đăng nhập
PUT/api/users/meCandidateCập nhật hồ sơ cá nhân (tên, SĐT, CV, kỹ năng)

c) Kết quả giao diện

Trang chỉnh sửa hồ sơ cá nhân ứng viên

UC05 Quản lý thông tin công ty

a) ERD liên quan

erDiagram USERS ||--o{ COMPANIES : "tạo" COMPANIES { int company_id PK varchar company_name text description varchar website varchar logo_url varchar address varchar status int created_by FK datetime created_at } USERS { int user_id PK varchar email varchar role }

UC05 truy cập bảng COMPANIES liên kết với USERS qua created_by. Mỗi Recruiter sở hữu duy nhất 1 công ty.

b) API Endpoints

MethodEndpointAuthMô tả
GET/api/companies/{id}PublicXem thông tin công ty theo ID
POST/api/companiesRecruiterTạo công ty mới (nếu chưa có)
PUT/api/companies/myRecruiterCập nhật thông tin công ty hiện tại

c) Kết quả giao diện

Trang cho phép Recruiter nhập/sửa: Tên công ty, Mô tả, Website, Logo URL, Địa chỉ.

UC06 Đăng tin tuyển dụng

a) ERD liên quan

erDiagram COMPANIES ||--o{ JOBS : "đăng" USERS ||--o{ JOBS : "tạo" JOBS ||--o{ JOB_SKILLS : "yêu cầu" SKILLS ||--o{ JOB_SKILLS : "được yêu cầu" JOBS { int job_id PK int company_id FK int recruiter_id FK varchar title text description text requirements decimal salary_min decimal salary_max varchar location varchar job_type varchar status date expiry_date datetime created_at } JOB_SKILLS { int job_id PK_FK int skill_id PK_FK }

UC06 tạo bản ghi mới trong JOBS (status='pending') và liên kết kỹ năng qua JOB_SKILLS. Tin cần Admin duyệt.

b) API Endpoints

MethodEndpointAuthMô tả
POST/api/jobsRecruiterĐăng tin tuyển dụng mới (status=pending, chờ admin duyệt)
GET/api/skillsPublicLấy danh sách kỹ năng để chọn tag

c) Kết quả giao diện

Form đăng tin gồm: Tiêu đề, Mô tả, Yêu cầu, Mức lương (min-max), Địa điểm, Loại hình, Kỹ năng (chọn tags), Ngày hết hạn.

UC07 Quản lý hồ sơ ứng viên

a) ERD liên quan

erDiagram JOBS ||--o{ APPLICATIONS : "nhận" USERS ||--o{ APPLICATIONS : "nộp" USERS ||--o{ APPLICATIONS : "duyệt" APPLICATIONS { int application_id PK int job_id FK int candidate_id FK varchar cv_url text cover_letter varchar status datetime applied_at int reviewed_by FK } JOBS { int job_id PK varchar title int recruiter_id FK } USERS { int user_id PK varchar full_name varchar email }

UC07 cho phép Recruiter xem danh sách APPLICATIONS theo job_id, sau đó cập nhật status (pending → shortlisted → interview → accepted/rejected).

b) API Endpoints

MethodEndpointAuthMô tả
GET/api/applications/job/{jobId}RecruiterDanh sách ứng viên cho tin tuyển dụng (chỉ chủ tin)
PUT/api/applications/{id}/statusRecruiterCập nhật trạng thái hồ sơ ứng viên

c) Kết quả giao diện

Recruiter xem danh sách ứng viên theo từng tin đăng. Có thể click xem CV, đổi trạng thái (Mời phỏng vấn, Chấp nhận, Từ chối).

UC08 Quản lý bài đăng tuyển dụng

a) ERD liên quan

erDiagram USERS ||--o{ JOBS : "quản lý" COMPANIES ||--o{ JOBS : "sở hữu" JOBS ||--o{ JOB_SKILLS : "yêu cầu" JOBS ||--o{ APPLICATIONS : "nhận" JOBS { int job_id PK int company_id FK int recruiter_id FK varchar title varchar status date expiry_date datetime created_at }

UC08 cho phép Recruiter xem tất cả JOBS của mình, chỉnh sửa nội dung, đóng tin (status→closed) hoặc gia hạn (expiry_date +30 ngày).

b) API Endpoints

MethodEndpointAuthMô tả
GET/api/jobs/myRecruiterDanh sách tin tuyển dụng đã đăng
PUT/api/jobs/{id}RecruiterCập nhật nội dung tin tuyển dụng
PUT/api/jobs/{id}/closeRecruiterĐóng tin (status → closed)
PUT/api/jobs/{id}/renewRecruiterGia hạn tin +30 ngày (status → active)

c) Kết quả giao diện

Trang Quản lý tin tuyển dụng của Recruiter — Sửa, Đóng, Gia hạn, Xem ứng viên

UC09 Duyệt tin tuyển dụng (Admin)

a) ERD liên quan

erDiagram USERS ||--o{ JOBS : "duyệt" COMPANIES ||--o{ JOBS : "đăng" JOBS { int job_id PK int company_id FK int recruiter_id FK varchar title varchar status int approved_by FK datetime created_at } USERS { int user_id PK varchar role } COMPANIES { int company_id PK varchar company_name }

UC09: Admin xem các JOBS có status='pending' và cập nhật thành 'active' (duyệt) hoặc 'closed' (từ chối). Trường approved_by ghi lại admin nào đã duyệt.

b) API Endpoints

MethodEndpointAuthMô tả
GET/api/jobs/pendingAdminDanh sách tin chờ duyệt
PUT/api/jobs/{id}/approveAdminPhê duyệt tin (status → active)
PUT/api/jobs/{id}/rejectAdminTừ chối tin (status → closed)
PUT/api/jobs/bulk-approveAdminDuyệt hàng loạt
PUT/api/jobs/bulk-rejectAdminTừ chối hàng loạt

c) Kết quả giao diện

Trang Admin duyệt tin — Xem danh sách tin chờ, nút Duyệt/Từ chối cho từng tin

UC10 Quản lý người dùng (Admin)

a) ERD liên quan

erDiagram USERS { int user_id PK varchar email UK varchar full_name varchar phone varchar role varchar status datetime created_at }

UC10: Admin truy vấn toàn bộ bảng USERS (trừ admin), có thể khóa (status→suspended) hoặc xoá tài khoản vi phạm. Tìm kiếm tiếng Việt không dấu qua Vietnamese_CI_AI Collation.

b) API Endpoints

MethodEndpointAuthMô tả
GET/api/users?role=&search=AdminDanh sách người dùng (lọc theo role/tìm kiếm)
PUT/api/users/{id}/suspendAdminKhóa / Mở khóa tài khoản
DELETE/api/users/{id}AdminXóa tài khoản (không xóa được admin)
GET/api/admin/statsAdminThống kê tổng quan: số users, jobs, applications, companies

c) Kết quả giao diện

Trang Admin quản lý người dùng: Bảng danh sách (ID, Tên, Email, Role, Status), các nút Khóa/Mở khóa và Xóa với xác nhận email.
