1. UC01: Đăng nhập & Phân quyền

a. Sơ đồ dữ liệu (ERD)

erDiagram USERS { int user_id PK varchar email UK varchar password_hash varchar full_name varchar role varchar status datetime created_at }
(Sơ đồ mô tả cấu trúc bảng USERS phục vụ xác thực và phân quyền)

b. Danh sách API Endpoints

Phương thức Đường dẫn (Endpoint) Quyền (Auth) Mô tả chức năng
POST /api/auth/login Public Đăng nhập qua email/password, trả về Token JWT.
POST /api/auth/register Public Đăng ký tài khoản mới cho Ứng viên hoặc Nhà tuyển dụng.
POST /api/auth/google Public Xác thực qua Google OAuth và tự động tạo tài khoản.

c. Kết quả giao diện

Hình 1.1: Giao diện Trang Đăng nhập và Lựa chọn OAuth

2. UC02: Tìm kiếm việc làm

a. Sơ đồ dữ liệu (ERD)

erDiagram JOBS ||--o{ JOB_SKILLS : "yêu cầu" SKILLS ||--o{ JOB_SKILLS : "thuộc về" COMPANIES ||--o{ JOBS : "đăng tuyển" JOBS { int job_id PK varchar title varchar location decimal salary_min decimal salary_max varchar status }

b. Danh sách API Endpoints

Phương thức Đường dẫn (Endpoint) Quyền (Auth) Mô tả chức năng
GET /api/jobs Public Tìm kiếm và lọc tin tuyển dụng (Hỗ trợ tiếng Việt không dấu).

c. Kết quả giao diện

Hình 2.1: Giao diện tìm kiếm việc làm với bộ lọc đa năng

3. UC03: Xem chi tiết & Ứng tuyển

a. Sơ đồ dữ liệu (ERD)

erDiagram USERS ||--o{ APPLICATIONS : "nộp đơn" JOBS ||--o{ APPLICATIONS : "nhận hồ sơ" APPLICATIONS { int application_id PK int job_id FK int candidate_id FK varchar cv_url varchar status datetime applied_at }

b. Danh sách API Endpoints

Phương thức Đường dẫn (Endpoint) Quyền (Auth) Mô tả chức năng
GET /api/jobs/{id} Public Lấy thông tin chi tiết một công việc cụ thể.
POST /api/applications Candidate Gửi hồ sơ ứng tuyển (bao gồm link CV và Cover Letter).

c. Kết quả giao diện

Hình 3.1: Chi tiết công việc và nút Ứng tuyển nhanh

5. UC05: Quản lý thông tin công ty

Dành cho Nhà tuyển dụng cập nhật Profile doanh nghiệp.

Phương thứcEndpointAuthMô tả
PUT/api/companies/myRecruiterSửa tên, logo, địa chỉ và mô tả công ty.

6. UC06: Đăng tin tuyển dụng

erDiagram COMPANIES ||--o{ JOBS : "thuộc về" JOBS { varchar title text description varchar status }
Phương thứcEndpointAuthMô tả
POST/api/jobsRecruiterTạo tin tuyển dụng mới với trạng thái Chờ Duyệt (Pending).

7. UC07: Quản lý hồ sơ ứng viên

Nhà tuyển dụng xét duyệt danh sách các ứng viên đã nộp đơn vào bài đăng của mình.

Phương thứcEndpointAuthMô tả
GET/api/applications/job/{id}RecruiterXem danh sách chi tiết các ứng viên nộp đơn.
PUT/api/applications/{id}/statusRecruiterCập nhật trạng thái (Mời phỏng vấn, Chấp nhận, Từ chối).

8. UC08: Quản lý bài đăng tuyển dụng

Hình 8.1: Dashboard Nhà tuyển dụng - Quản lý danh sách tin đã đăng

9. UC09: Duyệt tin tuyển dụng (Admin)

Admin kiểm duyệt nội dung tin đăng trước khi cho phép hiển thị công khai trên hệ thống.

Phương thứcEndpointAuthMô tả
PUT/api/jobs/{id}/approveAdminPhê duyệt bài đăng (Pending -> Active).
Hình 9.1: Giao diện Admin phê duyệt bài viết hàng loạt

10. UC10: Quản lý người dùng (Admin)

Admin có quyền khóa (Suspend) các tài khoản vi phạm hoặc đăng tin spam.

Phương thứcEndpointAuthMô tả
PUT/api/users/{id}/suspendAdminTạm khóa hoặc mở khóa tài khoản người dùng.
GET/api/admin/statsAdminThống kê tổng lượng User, Job và Application.
