Báo Cáo Kỹ Thuật - Phần Backend Hệ Thống DevJobs

1. Tổng quan Kiến trúc

Phần Backend của hệ thống DevJobs được xây dựng theo mô hình RESTful Web API, đóng vai trò cung cấp các dịch vụ dữ liệu cho ứng dụng Frontend (ReactJS) thông qua các giao thức HTTP.

Hệ thống không sử dụng mô hình MVC (Model-View-Controller) truyền thống (nơi server render ra mã HTML). Thay vào đó, Backend chỉ xử lý logic nghiệp vụ, giao tiếp với cơ sở dữ liệu và trả về kết quả dưới định dạng JSON. Việc hiển thị giao diện người dùng hoàn toàn do Frontend đảm nhận (Single Page Application - SPA).

2. Công nghệ sử dụng

  • Ngôn ngữ lập trình: C# (C-Sharp)
  • Framework: ASP.NET Core 8.0 Web API
  • Cơ sở dữ liệu: Microsoft SQL Server
  • ORM (Object-Relational Mapping): Entity Framework Core (chế độ Database-First Scaffold)
  • Bảo mật & Xác thực:
    • JWT (JSON Web Token): Dùng để xác thực (Authentication) và phân quyền (Authorization) người dùng không lưu trạng thái (stateless).
    • BCrypt.Net-Next: Thư viện băm (hashing) mật khẩu một chiều an toàn, chống lại các cuộc tấn công Brute-force & Rainbow table.

3. Cấu trúc dự án Backend

Dự án DevJobsAPI được tổ chức theo các thành phần chính sau:

  • Controllers/: Nơi tiếp nhận các HTTP Request từ client, điều hướng xử lý và trả về HTTP Response. Phân tách logic theo từng đối tượng nghiệp vụ (Users, Jobs, Applications, Auth...).
  • Models/: Các lớp thực thể (Entities) ánh xạ trực tiếp với cấu trúc bảng trong CSDL SQL Server (được tự động sinh ra bởi EF Core Scaffold).
  • DTOs/ (Data Transfer Objects): Các lớp dữ liệu truyền tải. Dùng để tùy biến dữ liệu nhận vào (Request) và dữ liệu trả về (Response) để tránh lộ các trường nhạy cảm từ CSDL (như mật khẩu) và tối ưu hóa Payload.
  • Services/: Chứa các lớp xử lý nghiệp vụ chung, ví dụ JwtService để đảm nhiệm việc tạo và xác minh token bảo mật.
  • Program.cs & appsettings.json: File cấu hình hệ thống, chuỗi kết nối (Connection String), cấu hình CORS (cho phép frontend gọi API), và cấu hình Middleware bảo mật.

4. Chi tiết Danh sách Web API Endpoints

4.1. Xác thực (Authentication)

Method Endpoint Mô tả Phân quyền
POST /api/auth/register Đăng ký tài khoản mới (mã hóa mật khẩu) Public
POST /api/auth/login Đăng nhập, xác thực và sinh ra JWT Token Public

4.2. Quản lý Người dùng (Users)

Method Endpoint Mô tả Phân quyền
GET /api/users/me Lấy thông tin hồ sơ của người dùng đang đăng nhập Đăng nhập
PUT /api/users/me Cập nhật thông tin cá nhân hiện tại (tên, sđt, CV...) Candidate
GET /api/users Lấy và tìm kiếm danh sách người dùng Admin
PUT /api/users/{id}/suspend Khóa / Mở khóa tài khoản một người dùng Admin
DELETE /api/users/{id} Xóa hoàn toàn tài khoản khỏi CSDL Admin

4.3. Quản lý Đơn tuyển dụng (Jobs)

Method Endpoint Mô tả Phân quyền
GET /api/jobs Tìm kiếm và lọc các tin tuyển dụng đang "active" Public
GET /api/jobs/{id} Lấy chi tiết một tin tuyển dụng (kèm thông tin công ty) Public
POST /api/jobs Đăng tin tuyển dụng mới (trạng thái chờ duyệt "pending") Recruiter
GET /api/jobs/my Lấy danh sách các tin tuyển dụng nhà tuyển dụng đã đăng Recruiter
PUT /api/jobs/{id} Chỉnh sửa nội dung tin tuyển dụng (nếu chưa hết hạn) Recruiter
GET /api/jobs/pending Lấy danh sách các tin đang chờ kiểm duyệt Admin
PUT /api/jobs/{id}/approve Phê duyệt tin tuyển dụng chuyển sang "active" Admin

4.4. Quản lý Ứng tuyển (Applications)

Method Endpoint Mô tả Phân quyền
POST /api/applications Nộp hồ sơ ứng tuyển vào một tin tuyển dụng Candidate
GET /api/applications/my-applications Lịch sử các đơn đã ứng tuyển của mình Candidate
GET /api/applications/job/{jobId} Xem danh sách ứng viên nộp vào tin tuyển dụng cụ thể Recruiter
PUT /api/applications/{id}/status Cập nhật trạng thái ứng viên (shortlisted, interview, accepted, rejected) Recruiter

5. Chuẩn bị cho Giai đoạn Phân Tán (Distributed Database)

Lưu ý: Database của hệ thống đã được di chuyển từ (localdb) sang một SQL Server Service thực thụ (ví dụ: Developer/Express Edition) có mở cổng TCP/IP. Nhờ đó, Connection String trong appsettings.json hiện được cấu hình để có thể tiếp nhận kết nối qua môi trường mạng LAN/Internet. Điểm này rất quan trọng để phục vụ môn Cơ sở dữ liệu phân tán (Distributed Database), cho phép phân mảnh các bảng dữ liệu ra nhiều server vật lý khác nhau mà Backend vẫn có thể truy vấn tới.
